00 How To Use This Guide Intro
👨‍🏫 Teacher: You lead each section. Explain the concept in your own words first, then type the example code live in VS Code, and let your student guess/predict what will happen before running it. That's where learning clicks.
Session Flow

For each section: 1) Explain the concept casually (use analogies). 2) Show the code example. 3) Do the easy exercises together. 4) Let your student attempt medium exercises solo. 5) Do hard exercises as a team challenge.

Setup: Open VS Code, create a new file called lesson.html, and use the template: <!DOCTYPE html> <html> <body> <script> // code here </script> </body> </html>. Open it in Chrome and use F12 to open the Console.
01 JavaScript Basics ~20 min
What is JavaScript?

JavaScript is the language that makes websites do things. HTML is the skeleton, CSS is the clothes, JavaScript is the brain. Every button click, pop-up, live update — that's JavaScript.

alert()

Shows a pop-up box in the browser. It's the simplest way to display a message. We'll use it a lot at the beginning, then move to console.log for more professional debugging.

ExampleJS
// Shows a popup message
alert('Hello! Welcome to JavaScript.');

// Print to the browser console (F12 → Console)
console.log('Hello from the console!');

// Change the entire page content
document.body.innerHTML = '<h1>I changed the page!</h1>';
👨‍🏫 Teacher: Type the alert first, save the file, refresh Chrome. Watch their reaction. Then show console.log — explain "the console is where developers read messages, it's like a secret window."

02 Numbers & Math ~20 min
Basic Operators

JavaScript handles math just like a calculator. Addition +, Subtraction -, Multiplication *, Division /. Brackets control priority, just like in school.

Order of Operations

JS follows the same rules as maths: brackets first, then multiplication/division (left to right), then addition/subtraction. (2 + 3) * 4 equals 20, not 14.

The "Float Problem"

Computers store decimals imperfectly. Try 0.1 + 0.2 — you'll get 0.30000000000000004. For money, always work in cents to avoid this. Use Math.round() when needed.

ExampleJS
console.log(5 + 3);       // 8
console.log(10 - 4);      // 6
console.log(3 * 4);        // 12
console.log(15 / 3);       // 5

// Order of operations
console.log(2 + 3 * 4);    // 14
console.log((2 + 3) * 4);  // 20

// The float problem
console.log(0.1 + 0.2);    // 0.30000000000000004

// Money strategy (cents)
console.log((2095 + 799) / 100);  // 28.94
👨‍🏫 Tip: Ask the student: "Why do you think 0.1 + 0.2 isn't exactly 0.3?" Explain that computers use binary, which is like trying to write 1/3 as a decimal (0.333...) — it never quite ends.

03 Strings ~20 min
What is a String?

A string is just text. Any characters wrapped in quotes. You can use single quotes 'hello', double quotes "hello", or backticks `hello`. They mostly behave the same — but backticks have a superpower.

Concatenation vs Interpolation

Concatenation = gluing strings together with +. Interpolation = embedding values directly inside a string using backticks and ${ }. Interpolation is almost always cleaner — prefer it.

Coercion (the sneaky one)

When you use + between a string and a number, JavaScript converts the number into a string. So "5" + 3 gives you "53", not 8. This trips up a lot of beginners!

ExampleJS
// Three ways to write strings
'Hello'
"Hello"
`Hello`

// Concatenation
'Items (' + (2 + 3) + '): $' + (2095 + 799) / 100;

// Interpolation (Easier!)
`Items (${2 + 3}): $${(2095 + 799) / 100}`;

// Multi-line string (only with backticks)
`Hello
World`;

// Coercion
'5' + 3 // '53'
'5' - 3 // 2 (Math magic!)
👨‍🏫 Teacher: This is a great place to show "Why strings matter." Use the cart analogy: "We have numbers for math, but we need strings to show the user a pretty message like 'You have 5 items in your cart'."

04 Variables ~20 min
What is a Variable?

A variable is a named box for storing a value. You give it a name, put something inside, and use that name whenever you need the value. Simple.

KeywordCan change?When to use
constNo — it's lockedDefault choice. Use for values that won't change.
letYes — flexibleWhen the value will change (e.g. a cart counter).
varYes — but messyOld style, has scope issues. Avoid in modern code.
ExampleJS
// const — can't be reassigned
const pi = 3.14159;
console.log(pi);             // 3.14159

// let — can be reassigned
let score = 0;
console.log(score);          // 0
score = 10;
console.log(score);          // 10

// Shorthand update operators
score += 5;                  // same as score = score + 5
score++;                     // same as score = score + 1
👨‍🏫 Teacher: Use the "Storage Box" analogy. const is a box that's taped shut after you put something in. let is a box with a lid you can keep opening and changing.

05 Booleans & Comparisons ~20 min
Booleans

A boolean has only two possible values: true or false. Think of it like a light switch — on or off.

OperatorMeaningExampleResult
===Strict equals5 === 5true
!==Not equal5 !== 3true
>Greater than10 > 5true
<Less than3 < 1false
>=Greater or equal18 >= 18true
<=Less or equal5 <= 4false
ExampleJS
console.log(5 === 5);        // true
console.log(5 === '5');      // false (Strict equals checks the type too!)
console.log(10 > 3);         // true
console.log(18 >= 18);       // true

// typeof operator
console.log(typeof 'hello'); // 'string'
console.log(typeof 5);       // 'number'
console.log(typeof true);    // 'boolean'
👨‍🏫 Teacher: Use the "Strict Equals" (===) as the gold standard. Explain that == exists but is a "loose" cousin that causes bugs by trying to be too helpful.

06 If-Statements ~25 min
👨‍🏫 Teacher: This is where logic begins! Use real-world examples like "If it's raining, take an umbrella."
The if-statement

An if-statement lets your code make decisions. The code in { } only runs if the condition in ( ) is truthy.

StructureJS
if (condition) {
    // runs when condition is true
} else {
    // runs when condition is false
}
ExampleJS
const age = 15;

if (age >= 18) {
    console.log('You can drive!');
} else {
    console.log('Too young!');
}
👨‍🏫 Tip: Be sure to show them the different outputs. Change the age to 20 and watch the message change. This is the first time the code "thinks".

07 Logical Operators Deep Dive ~15 min
&& (AND) | || (OR) | ! (NOT)

Operators allow you to combine conditions. AND requires everything to be true. OR requires only one. NOT flips the truth.

Truth TableJS
true && true   // true
true && false  // false
true || false  // true
false || false // false
!true           // false
👨‍🏫 Teacher: Use the "Club Entry" analogy. AND = "You need a ticket AND you must be 18+". OR = "You get in if you're a VIP OR if you're with a VIP."

08 Rock Paper Scissors Project ~40 min
👨‍🏫 Teacher: Climax Project! Build this step-by-step. Talk through the logic before typing. This combines random numbers, variables, and if-statements.
The Game Logic

1. Computer picks random number. 2. Map 0-1/3 to Rock, 1/3-2/3 to Paper, etc. 3. Compare with player choice. 4. Log the result.

Step 1: Computer MoveJS
const randomNumber = Math.random();
let computerMove = '';

if (randomNumber >= 0 && randomNumber < 1/3) {
    computerMove = 'rock';
} else if (randomNumber >= 1/3 && randomNumber < 2/3) {
    computerMove = 'paper';
} else {
    computerMove = 'scissors';
}
console.log(computerMove);
Step 2: Comparison (Rock Button)JS
if (computerMove === 'rock') {
    result = 'Tie.';
} else if (computerMove === 'paper') {
    result = 'You lose.';
} else {
    result = 'You win.';
}
alert(`You picked rock. Computer picked ${computerMove}. ${result}`);
