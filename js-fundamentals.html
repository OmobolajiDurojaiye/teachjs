01 JavaScript Basics ⏱ ~20 min
What is JavaScript?

JavaScript is the language that makes websites do things. HTML is the skeleton, CSS is the clothes, JavaScript is the brain. Every button click, pop-up, live update — that's JavaScript.

alert()

Shows a pop-up box in the browser. It's the simplest way to display a message. We'll use it a lot at the beginning, then move to console.log for more professional debugging.

ExampleJS
// Shows a popup message
alert('Hello! Welcome to JavaScript.');

// Print to the browser console (F12 → Console)
console.log('Hello from the console!');

// Change the entire page content
document.body.innerHTML = '<h1>I changed the page!</h1>';
👨‍🏫 Tip: Type the alert first, save the file, refresh Chrome. Watch her reaction. Then show console.log — explain "the console is where developers read messages, it's like a secret window."

02 Numbers & Math ⏱ ~20 min
Basic Operators

JavaScript handles math just like a calculator. Addition +, Subtraction -, Multiplication *, Division /. Brackets control priority, just like in school.

Order of Operations (BODMAS)

JS follows the same rules as maths: brackets first, then multiplication/division (left to right), then addition/subtraction. (2 + 3) * 4 equals 20, not 14.

The Float Problem

Computers store decimals imperfectly. Try 0.1 + 0.2 — you'll get 0.30000000000000004. For money, always work in cents to avoid this. Use Math.round() when needed.

ExamplesJS
console.log(5 + 3);       // 8
console.log(10 - 4);      // 6
console.log(3 * 4);        // 12
console.log(15 / 3);       // 5

// Order of operations
console.log(2 + 3 * 4);    // 14 (not 20!)
console.log((2 + 3) * 4);  // 20

// Percentages
console.log(200 * 0.1);     // 10% of 200 = 20

// The float problem
console.log(0.1 + 0.2);    // 0.30000000000000004 😱

// Fix with rounding
console.log(Math.round(2.7));   // 3
console.log(Math.round(2.3));   // 2

// Money in cents strategy
console.log((2095 + 799) / 100);  // $28.94 ✅

03 Strings ⏱ ~20 min
What is a String?

A string is just text. Any characters wrapped in quotes. You can use single quotes 'hello', double quotes "hello", or backticks `hello`. They mostly behave the same — but backticks have a superpower.

Concatenation vs Interpolation

Concatenation = gluing strings together with +. Interpolation = embedding values directly inside a string using backticks and ${ }. Interpolation is almost always cleaner — prefer it.

Coercion (the sneaky one)

When you use + between a string and a number, JavaScript converts the number into a string. So "5" + 3 gives you "53", not 8. This trips up a lot of beginners!

ExamplesJS
// Three ways to write strings
console.log('Single quotes');
console.log("Double quotes");
console.log(`Backticks`);

// Concatenation (old way)
const name = 'Zara';
console.log('Hello, ' + name + '!');   // Hello, Zara!

// Interpolation with backticks (modern way ✅)
const age = 22;
console.log(`Hello, ${name}! You are ${age} years old.`);

// Math inside interpolation
console.log(`2 + 2 = ${2 + 2}`);          // 2 + 2 = 4

// Coercion — watch out!
console.log("5" + 3);       // "53" 😱 (not 8!)
console.log("5" - 3);       // 2 (- doesn't concatenate)

// When to use double quotes
console.log("I'm learning JS");  // apostrophe inside fine

04 Variables ⏱ ~20 min
What is a Variable?

A variable is a named box for storing a value. You give it a name, put something inside, and use that name whenever you need the value. Simple.

KeywordCan change?When to use
constNo — it's lockedDefault choice. Use for values that won't change.
letYes — flexibleWhen the value will change (e.g. a cart counter).
varYes — but messyOld style, has scope issues. Avoid in modern code.
ExamplesJS
// const — can't be reassigned
const pi = 3.14159;
console.log(pi);             // 3.14159

// let — can be reassigned
let score = 0;
console.log(score);          // 0
score = 10;
console.log(score);          // 10

// Shorthand update operators
score = score + 5;          // Same as score += 5
score += 5;                  // score is now 20
score++;                     // score is now 21
console.log(score);          // 21

// typeof — check what type a variable is
console.log(typeof score);   // "number"
console.log(typeof 'hello'); // "string"
console.log(typeof true);    // "boolean"
⚠️ Common mistake: Trying to change a const gives an error: "Assignment to constant variable". If you need to change a value later, use let.

05 Booleans & Comparisons ⏱ ~20 min
Booleans

A boolean has only two possible values: true or false. Think of it like a light switch — on or off. Most of your if-statements will use booleans.

Comparison Operators

These compare two values and return a boolean (true or false).

OperatorMeaningExampleResult
===Strict equals5 === 5true
!==Not equal5 !== 3true
>Greater than10 > 5true
<Less than3 < 1false
>=Greater or equal18 >= 18true
<=Less or equal5 <= 4false
==Loose equals (avoid!)"5" == 5true (BAD)
Truthy & Falsy

Every value in JS is either "truthy" or "falsy" when used in a condition. Falsy values: false, 0, '' (empty string), NaN, undefined, null. Everything else is truthy.

ExamplesJS
console.log(5 === 5);        // true
console.log(5 === "5");      // false (different types!)
console.log(10 > 3);         // true
console.log(18 >= 18);       // true

// Ternary operator (shorthand if-else)
// condition ? 'if true' : 'if false'
const canDrive = 20 >= 18 ? 'Yes!' : 'No.';
console.log(canDrive);       // Yes!

// Truthy / falsy check
const cart = 0;
console.log(Boolean(cart));   // false (0 is falsy)
console.log(Boolean(5));      // true
console.log(Boolean(''));     // false (empty string)
console.log(Boolean('hi'));   // true

06 If-Statements & Control Flow ⏱ ~25 min
The if-statement

An if-statement lets your code make decisions. "If this thing is true, run this code. Otherwise, run that code." The code in { } only runs if the condition in ( ) is truthy.

Block Scope

Variables declared inside { } with const or let only exist inside those braces. This is called block scope. If you need a variable in multiple branches, declare it outside the if-statement first.

StructureJS
if (condition) {
    // runs when condition is true
} else if (anotherCondition) {
    // runs when first is false, this is true
} else {
    // runs when ALL conditions are false
}
Full ExampleJS
const driver = 'Amara';
const age = 17;

if (age >= 18) {
    console.log(`${driver}, you can drive!`);
    document.body.innerHTML = `${driver} can drive ✅`;
} else {
    console.log(`${driver}, you cannot drive yet.`);
    document.body.innerHTML = `${driver} cannot drive ❌`;
}

// Multiple branches (else if)
const score = 75;

if (score >= 90) {
    console.log('Grade: A');
} else if (score >= 70) {
    console.log('Grade: B');
} else if (score >= 50) {
    console.log('Grade: C');
} else {
    console.log('Grade: F — retake required');
}

// Scope example — declare OUTSIDE for access in both branches
let message = '';

if (score >= 50) {
    message = 'Passed!';
} else {
    message = 'Failed.';
}

console.log(message);  // accessible here ✅

07 Logical Operators ⏱ ~15 min
&& (AND)

Returns true only if both conditions are true. Like a door that requires both a key AND a code — one is not enough.

|| (OR)

Returns true if at least one condition is true. Like a door that opens with a key OR a fingerprint — either works.

! (NOT)

Flips a boolean. true becomes false, false becomes true. Useful for "if NOT this condition".

ExamplesJS
const age = 20;
const hasLicense = true;

// AND: both must be true
if (age >= 18 && hasLicense) {
    console.log('You can drive!');
}

// OR: at least one must be true
const isWeekend = false;
const isHoliday = true;

if (isWeekend || isHoliday) {
    console.log('Day off!');
}

// NOT: flip it
const isLoggedIn = false;

if (!isLoggedIn) {
    console.log('Please log in first.');
}

// Combined example
console.log(true && false);   // false
console.log(true || false);   // true
console.log(!true);            // false

08 Project: Rock Paper Scissors 🎮 ⏱ ~40 min
👨‍🏫 This is the climax! Build this together from scratch. First, talk through the steps in English before writing any code. Let her write as much as possible. You guide, she types.
Step 1: Plan in English First

Before writing a single line of code, say out loud: "When I click Rock, what needs to happen?" Get her to describe it step by step. This is how real developers think.

The 4 steps: 1) Player picks a move (click) → 2) Computer picks randomly → 3) Compare both moves → 4) Show the result
Step 2: Math.random() for the Computer

Math.random() gives a decimal between 0 (inclusive) and 1 (exclusive). Divide the range into thirds: 0–1/3 = Rock, 1/3–2/3 = Paper, 2/3–1 = Scissors.

Computer Move LogicJS
const randomNumber = Math.random();
console.log(randomNumber);  // e.g. 0.73451...

let computerMove = '';

if (randomNumber >= 0 && randomNumber < 1/3) {
    computerMove = 'rock';
} else if (randomNumber >= 1/3 && randomNumber < 2/3) {
    computerMove = 'paper';
} else {
    computerMove = 'scissors';
}

console.log(`Computer picked: ${computerMove}`);
Step 3: Determine the Result (Player picks Rock)

Now compare the player's move to the computer's move. Rock vs Rock = Tie. Rock vs Paper = Lose. Rock vs Scissors = Win.

Full Rock Button LogicJS
// ---- Inside the onclick for Rock button ----

const randomNumber = Math.random();
let computerMove = '';

if (randomNumber < 1/3) {
    computerMove = 'rock';
} else if (randomNumber < 2/3) {
    computerMove = 'paper';
} else {
    computerMove = 'scissors';
}

let result = '';

if (computerMove === 'rock') {
    result = 'Tie!';
} else if (computerMove === 'paper') {
    result = 'You lose 😞';
} else {
    result = 'You win! 🎉';
}

alert(`You picked rock. Computer picked ${computerMove}. ${result}`);
Step 4: Complete the Other Buttons

The Paper and Scissors buttons have the same computer move logic. Only the result section changes. Let her write the Paper and Scissors button logic herself after you do Rock together.

HTML Buttons StructureHTML
<!DOCTYPE html>
<html>
<body>
    <h2>Rock Paper Scissors</h2>
    
    <button onclick="/* rock logic here */>🪨 Rock</button>
    <button onclick="/* paper logic here */>📄 Paper</button>
    <button onclick="/* scissors logic here */>✂️ Scissors</button>
</body>
</html>
