00 How To Use This Guide Read first
👨‍🏫 You are the teacher. Lead each section. Explain the concept in your own words first — use the analogies here. Then type the code live in VS Code. Ask your student to guess the output before you run it. That moment of prediction → verification is where learning really clicks.
Session Flow — Do This For Every Section

1. Explain the concept casually using the analogy provided. Don't just read — talk to your student.
2. Open VS Code, type the code example together. Let them watch.
3. Run it in the console (F12). Ask: "what do you think this will print?"
4. Do the Easy exercises together, out loud.
5. Medium exercises — they try, you watch. Help only if they're stuck for 2+ minutes.
6. Hard exercises — team effort. Discuss, think together, build it.

01 JavaScript Basics ~20 min
~8 min concept ~12 min exercises
🧠 The Analogy: A website has three layers. HTML is the skeleton — it gives structure. CSS is the clothes — it makes it look good. JavaScript is the brain — it makes things actually happen. Every button click, notification, form validation, live update — that's JavaScript thinking.
What is JavaScript?

JavaScript is a programming language that runs inside the browser. You write instructions, the browser follows them. The browser understands three languages: HTML (structure), CSS (style), and JavaScript (logic and behaviour).

alert()

The simplest way to display something. It pops open a box in the browser with a message. We'll use this at the start because it's very visible — but real developers use console.log() instead, because alerts are annoying to users.

console.log()

This prints a message to the browser's developer console. Open it by pressing F12 and clicking the Console tab. Think of the console as a developer's secret window — users can't see it, but you can. You'll use this constantly while building things.

document.body.innerHTML

This changes the entire visible content of a web page. It's JavaScript talking to the HTML. You're reaching into the page and replacing what's there with new content. It's like a magic eraser and pen in one.

Syntax

Syntax is the grammar of a programming language. Just like English has rules (a sentence ends with a period), JavaScript has rules too. Most statements end with a semicolon ;. Strings must be wrapped in quotes. Breaking the rules = the code doesn't run.

Example — Your first JSJS
// Two forward slashes = a comment. JS ignores this line.
// Comments are notes for humans, not the computer.

// 1. A pop-up alert
alert('Hello! Welcome to JavaScript.');

// 2. Print to the developer console (F12 → Console tab)
console.log('Hello from the console!');

// 3. Change the entire page content
document.body.innerHTML = '<h1>I changed the page!</h1>';

// 4. Math works directly
console.log(5 + 3);
👨‍🏫 Teacher: Type the alert() first, save the file, refresh Chrome. Watch their reaction. Then comment out the alert and show console.log(). Say: "The console is a developer's secret window — users never see this, it's only for us." Then show document.body.innerHTML — it's always impressive when the whole page changes with one line.

02 Numbers & Math ~20 min
~8 min concept ~12 min exercises
🧮 The Analogy: JavaScript is basically a calculator that also speaks English. Numbers work exactly how you'd expect — the only weird part is decimals, because computers count in binary (0s and 1s), which makes some fractions imprecise. We work around it like pros.
Basic Math Operators

+ Addition  ·  - Subtraction  ·  * Multiplication  ·  / Division  ·  % Remainder (Modulus)

Order of Operations (BODMAS/PEMDAS)

JavaScript follows the same maths rules you learned in school. Brackets first. Then multiplication and division (left to right). Then addition and subtraction. When in doubt — use brackets!

Example: 2 + 3 * 4 = 14 (not 20, because * happens first). (2 + 3) * 4 = 20 (brackets force + to go first).

Integers vs Floats

Integer = whole number (1, 5, 100). Float = decimal number (3.14, 0.5, 99.99). JavaScript treats them both as the type number — no distinction needed in your code.

The Float Problem 🤯

Try running console.log(0.1 + 0.2). You'll get 0.30000000000000004 — not 0.3. This is because computers store decimals in binary, and some decimals (like 0.1) can't be stored perfectly, just like 1/3 in decimal is 0.3333... forever.

The fix for money: Never work with decimal dollars. Work in cents instead. $20.95 = 2095 cents. Add the cents, then divide by 100 at the end.

Math.round() and Math.random()

Math.round(2.7) → 3. Rounds to the nearest whole number.
Math.random() → a random decimal between 0 and 1 (like 0.4839...). We'll use this in the Rock Paper Scissors game.

ExampleJS
// Basic math
console.log(5 + 3);           // 8
console.log(10 - 4);          // 6
console.log(3 * 4);            // 12
console.log(15 / 3);           // 5
console.log(10 % 3);           // 1 (remainder after 10 ÷ 3)

// Order of operations
console.log(2 + 3 * 4);      // 14 (not 20)
console.log((2 + 3) * 4);    // 20

// The float problem
console.log(0.1 + 0.2);        // 0.30000000000000004 😬

// The cents strategy (for money)
console.log((2095 + 799) / 100);  // 28.94 ✓

// Math helpers
console.log(Math.round(4.7));    // 5
console.log(Math.round(4.2));    // 4
console.log(Math.random());      // something between 0 and 1
👨‍🏫 Teacher: The float problem always gets a good reaction. Ask: "What do you think 0.1 + 0.2 is?" — they'll say 0.3. Run it and watch the surprise. Then explain the binary analogy: "It's like trying to write 1/3 as a decimal. You'd write 0.333... forever. Computers have the same problem with some decimals." Then show the cents fix — explain this is how Paystack, Flutterwave, and every payment company in the world actually handles money.

03 Strings ~20 min
~8 min concept ~12 min exercises
💬 The Analogy: Numbers are for calculating. Strings are for communicating. When you want JavaScript to talk to a user — show a message, display a name, print a receipt — you need strings. A string is just text wrapped in quotes. Simple.
What is a String?

A string is any piece of text. Wrap it in quotes and it becomes a string. You can use three kinds of quotes — they mostly behave the same, but backticks have a superpower called interpolation.

Quote TypeSyntaxWhen to use
Single quotes'Hello'Default. Use for most strings.
Double quotes"Hello"When your string contains a single quote: "I'm learning"
Backticks`Hello`When you need to embed variables or expressions inside the string (interpolation). Also works for multi-line strings.
Concatenation

Using + to join strings together. It works, but it's messy and easy to forget spaces.

Interpolation (Prefer This!)

Embedding a variable or expression directly inside a backtick string using ${ }. This is cleaner, easier to read, and less error-prone than concatenation. Use this by default whenever you need variables inside strings.

Type Coercion (The Sneaky One)

When you use + between a string and a number, JavaScript converts the number into a string automatically. So "5" + 3 gives you "53", not 8. This trips everyone up at least once. The rule: if JS sees a string and a +, it joins, it doesn't add.

ExampleJS
// The three ways to write strings
'Single quotes'
"Double quotes — useful when there's an apostrophe"
`Backticks — the powerful ones`

// Concatenation (old way)
console.log('My name is ' + 'Ada');
console.log('Items (' + (2 + 3) + '): $' + (2095 + 799) / 100);

// Interpolation (better way — uses backticks)
console.log(`My name is ${'Ada'}`);
console.log(`Items (${2 + 3}): $${(2095 + 799) / 100}`);

// Multi-line — only backticks can do this!
console.log(`Line one
Line two
Line three`);

// Coercion — the tricky one
console.log('5' + 3);   // '53' — joined as text!
console.log('5' - 3);   // 2 — subtraction forces it to be a number
👨‍🏫 Teacher: Start by asking "what do you think '5' + 3 gives?" They'll probably say 8. Run it — show it gives "53". Then show '5' - 3 = 2. Explain: the + is doing double duty — it's both math addition AND text joining. When JS sees a string, it picks text-joining. But - only works for math, so JS is forced to convert '5' to a number. Then show interpolation and ask your student which is cleaner — they'll pick backticks every time.

04 Variables ~20 min
~8 min concept ~12 min exercises
📦 The Analogy: A variable is like a labelled box. You put a value in the box and give the box a name. Whenever you need that value, you just use the box's name. const is a box you tape shut — you can look inside but never change what's in it. let is a box you can keep opening and swapping things out.
What is a Variable?

A variable stores a value so you can use it later — and reference it by name instead of repeating the value everywhere. This makes code smarter, reusable, and easier to change.

KeywordCan change?When to useExample
const❌ No — it's lockedDefault. Use this unless you know it'll change.const pi = 3.14
let✅ Yes — flexibleWhen the value will change (e.g. a score, a cart count).let score = 0
var✅ Yes — but messyOld way. Has scope issues. Don't use in new code.var x = 5 (avoid)
Naming Rules

✅ Can contain letters, numbers, $, and _
✅ Use camelCase: myName, cartTotal, isLoggedIn
❌ Cannot start with a number: 1name = error
❌ Cannot use reserved words: let let = 5 = error
❌ Cannot have spaces: my name = error

Shorthand Update Operators

score = score + 1 ← long way
score += 1 ← shorthand (same thing)
score++ ← even shorter (only adds 1)

Also: -=, *=, /=. These all update a variable in place.

ExampleJS
// const — locked, can't reassign
const pi = 3.14159;
console.log(pi);              // 3.14159
// pi = 3.14;               // ❌ Error: Assignment to constant variable

// let — flexible, can reassign
let score = 0;
console.log(score);          // 0
score = 10;                  // reassign ✓
console.log(score);          // 10

// Shorthand operators
score += 5;                  // same as score = score + 5 → 15
score++;                     // same as score = score + 1 → 16
score *= 2;                  // same as score = score * 2 → 32
console.log(score);          // 32

// Using variables in interpolation
const name = 'Ada';
const items = 3;
console.log(`Hello ${name}, you have ${items} items in your cart.`);
👨‍🏫 Teacher: Show the error you get when trying to reassign a const — it's a great teaching moment. Then demonstrate shorthand operators with a live score counter: start at 0, use score++ a few times, log it each time. Connect it to games ("this is literally how a game score works"). Naming: share the golden rule — names should describe what the variable contains. x is bad, cartTotal is good.

05 Booleans & Comparisons ~20 min
~8 min concept ~12 min exercises
💡 The Analogy: A boolean is a light switch. It's either ON (true) or OFF (false). That's it. Only two options. Booleans are the language computers use to make decisions — "Is the user logged in? Is the cart empty? Is the password correct?" — yes or no, true or false.
Booleans

A boolean is a data type with only two possible values: true or false. No quotes, no numbers — just the words true and false.

OperatorMeaningExampleResult
===Strict equal (checks value AND type)5 === 5true
!==Strict not equal5 !== 3true
==Loose equal (type-converts, avoid!)5 == '5'true 😬
>Greater than10 > 5true
<Less than3 < 1false
>=Greater or equal18 >= 18true
<=Less or equal5 <= 4false
=== vs == (Important!)

=== (triple equals) is the strict version. It checks both the value AND the data type. This is almost always what you want.
== (double equals) is the loose version — it converts types before comparing, which causes unexpected bugs. Always use ===.

Truthy & Falsy Values

In JavaScript, not only true and false are booleans — every value has a "truthy" or "falsy" quality when used in a condition. Falsy means JS treats it like false. Everything else is truthy.

false
the boolean false
0
the number zero
''
empty string
null
intentional nothing
undefined
never assigned
NaN
Not a Number
ExampleJS
// Basic booleans
console.log(true);             // true
console.log(false);            // false

// Comparison operators — they return a boolean
console.log(5 === 5);         // true
console.log(5 === '5');       // false (different types!)
console.log(5 == '5');        // true (loose! converts before checking)
console.log(10 > 3);          // true
console.log(18 >= 18);        // true

// typeof — inspect what type a value is
console.log(typeof 'hello');  // 'string'
console.log(typeof 5);        // 'number'
console.log(typeof true);     // 'boolean'

// Why 5 === '5' is false
console.log(typeof 5);        // 'number'
console.log(typeof '5');      // 'string' — different types!
👨‍🏫 Teacher: Show the === vs == difference with 5 == '5' (true) and 5 === '5' (false). Explain: "double equals tries to be helpful and converts types, but this causes bugs you don't expect. Triple equals never converts — what you see is what you get. Always use ===." Then show typeof — it's a detective tool. Ask: "How would you prove to JavaScript that 5 and '5' are different? typeof!" This also sets up the falsy concept nicely.

06 If-Statements & Control Flow ~25 min
~10 min concept ~15 min exercises
🚦 The Analogy: An if-statement is a traffic light. If green, go. If yellow, slow down. If red, stop. Your code uses the same logic — check a condition, then decide what to do based on the answer. This is where your code starts to actually think.
The if-statement

An if-statement lets your code make decisions. The code inside { } only runs if the condition in ( ) is truthy. If it's not, that block is skipped entirely.

else and else if

else runs when all previous conditions are false — it's your catch-all fallback. else if lets you check another condition before falling back. You can chain as many else if blocks as you need.

Scope

Variables created inside { } only exist inside those braces. This is called block scope. If you create a variable inside an if block, you can't use it outside. To use a variable in multiple branches, create it before the if-statement.

Ternary Operator — The One-Liner

A shortcut for simple if-else that fits on one line. Syntax: condition ? value_if_true : value_if_false. Great for simple cases, but don't overuse it — complex ternaries are hard to read.

StructureJS
if (condition) {
    // runs when condition is truthy
} else if (anotherCondition) {
    // runs if first failed, but this is truthy
} else {
    // runs when everything above is false
}
Example — Age CheckJS
const age = 15;

if (age >= 18) {
    console.log('You can drive!');
} else if (age >= 16) {
    console.log('You can get a learner permit.');
} else {
    console.log('Too young for any of that!');
}
Example — Ternary OperatorJS
// Long way (if-else)
let status;
if (age >= 18) {
    status = 'adult';
} else {
    status = 'minor';
}

// Short way (ternary)
const status2 = age >= 18 ? 'adult' : 'minor';
console.log(status2);
👨‍🏫 Teacher: Use the traffic light analogy first. Then build the age example live — change the number and run it repeatedly: "See how the output changes based on the value? The code is thinking." Show scope by creating a variable inside the if block and trying to log it outside — it'll throw an error. This is an important lesson. Leave the ternary for last — some students find it elegant, others find it confusing at first. Both are fine.

07 Logical Operators ~15 min
~6 min concept ~9 min exercises
🎭 The Analogy: AND (&&) = The bouncer at the VIP section. "You need a wristband AND you must be on the list." Both conditions must be true. OR (||) = The door at the side entrance. "You get in if you're a VIP OR if you're with a VIP." Either condition works. NOT (!) = A flip. "Not raining" means it's sunny.
&& (AND)

Returns true only if both sides are true. If either side is false, the whole thing is false.

|| (OR)

Returns true if either side is true. Only returns false if both sides are false.

! (NOT)

Flips a boolean. !true = false. !false = true. Useful for checking the opposite of a condition without writing a whole comparison.

Truth TableJS
// AND — both must be true
console.log(true  && true);   // true
console.log(true  && false);  // false
console.log(false && false);  // false

// OR — either can be true
console.log(true  || false);  // true
console.log(false || false);  // false

// NOT — flips it
console.log(!true);            // false
console.log(!false);           // true

// Real example: Club entry
const age = 20;
const hasTicket = true;
if (age >= 18 && hasTicket) {
    console.log('Welcome to the club!');
}
👨‍🏫 Teacher: Walk through each operator with real-world examples. For AND: "To open your bank account online, you need your password AND your OTP." For OR: "You can pay with your card OR cash." For NOT: "If not logged in, show the login page." Then ask your student to predict the truth table before running it — this builds the mental model fast.

Randomization Strategy

Math.random() gives a number from 0 (inclusive) to 1 (exclusive). We need to map that to one of three moves. Divide the 0–1 range into thirds:

0 to 1/3 → Rock  |  1/3 to 2/3 → Paper  |  2/3 to 1 → Scissors

    computerMove = 'Paper';

    computerMove = 'Scissors';

// Put this inside a <script> tag in your HTML
function playGame(playerMove) {
  // Step 1: Get computer's move
  const randomNumber = Math.random();
  let computerMove = '';

  if (randomNumber < 1/3) {
    computerMove = 'Rock';
  } else if (randomNumber < 2/3) {
    computerMove = 'Paper';
  } else {
    computerMove = 'Scissors';
  }

  // Step 2: Determine result
  let result = '';
  if (playerMove === computerMove) {
    result = "It's a tie!";
  } else if (
    (playerMove === 'Rock'     && computerMove === 'Scissors') ||
    (playerMove === 'Paper'    && computerMove === 'Rock')     ||
    (playerMove === 'Scissors' && computerMove === 'Paper')
  ) {
    result = 'You win! 🎉';
  } else {
    result = 'Computer wins! 🤖';
  }

  // Step 3: Show the result
  alert(`You: ${playerMove} | Computer: ${computerMove}\n${result}`);
}
